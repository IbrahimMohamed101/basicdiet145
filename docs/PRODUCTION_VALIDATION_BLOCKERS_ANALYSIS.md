## Infrastructure Notes

- `npm run test:addon-credit-allocation` is pure in-process JavaScript and does not require MongoDB.
- `npm run validate:backend` uses `mongodb-memory-server` for `tests/oneTimeMenuCatalog.test.js`.
- `npm run test:addon-credit-lifecycle` and `npm run test:addon-dashboard-mobile-parity` require `mongodb-memory-server` replica sets because they exercise transactional MongoDB paths.
- The premium lifecycle wrapper uses the repository's test harness and stops at `npm run test:integration` in the current failure state.
- No live payment provider, production database, dashboard repo, or Flutter repo is required for these failures.
